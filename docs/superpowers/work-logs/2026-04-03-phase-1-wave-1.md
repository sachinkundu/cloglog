## Learnings & Issues

1. **Agents pushed as user identity** — wt-board and wt-frontend pushed and created PRs as sachinkundu instead of the bot. Only wt-mcp got it right. Root cause: CLAUDE.md had no bot identity instructions, and the worktree CLAUDE.md template didn't include them. **Fixed:** Added "Git Identity & PRs" section to CLAUDE.md, updated create-worktree.sh template, and added to launch prompts.

2. **Frontend PR had no tests** — wt-frontend shipped components without component tests. Only App.test.tsx and useSSE.test.ts existed. Had to send a second agent to add tests after PR review. **Fixed:** Added "every PR must include tests" to Agent Learnings in CLAUDE.md.

3. **Frontend PR had no screenshots** — For UI work, reviewers need to see what was built. **Fixed:** Added to Agent Learnings.

4. **MCP server only partially completed** — wt-mcp created the server scaffold and register_agent tool but didn't implement all 9 tools from the plan. Heartbeat timer and most tools are still missing. **Needs:** Wave 2 or follow-up work to complete MCP tools.

5. **Zellij layout replaced existing tabs** — Using `--layout` with `new-tab` replaced the user's existing session. **Fixed:** Switched to `new-tab` + `new-pane` approach within current zellij session.

6. **Shell expansion broke agent prompts** — Passing prompts with `${BOT_TOKEN}` through bash expanded them to empty strings. **Fixed:** Write prompts to files and read with `$(cat prompt.txt)` inside single quotes.
