### Worktree Agent Process
- **CONTRACT.yaml committed to PR** — agents committed the local reference file. Fixed by adding to .gitignore and updating create-worktree.sh.
- **Enter key in zellij** — `zellij action write 10` doesn't submit input in the claude REPL. User must manually press Enter in each pane to kick agents. Needs investigation.
- **Agents don't detect PR merge** — after PR is merged, agents sit idle instead of marking tasks done, unregistering, and exiting. The `/loop` only watches for review comments. Agent prompts need to instruct checking `gh pr view --json state` for merge detection.
- **Agent prompts as files** — writing AGENT_PROMPT.md works well. Agents read the file and execute tasks in order. CLAUDE.md in the worktree provides context.
