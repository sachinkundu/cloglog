## The fixes

### T-250 — dev toolchain in the bootstrap

`.cloglog/on-worktree-create.sh:18` was running `uv sync 2>/dev/null || true`, pulling only `[project.dependencies]`. Fresh worktree `.venv`s shipped without `pytest`, so `uv run pytest …` silently fell through to a pyenv shim and failed with a misleading `ModuleNotFoundError: No module named 'respx'` — one level removed from the actual root cause. T-247 and T-249 both hit this independently.

Fix:

```bash
uv sync --extra dev || true
[[ -x "$WORKTREE_PATH/.venv/bin/pytest" ]] || echo "WARN: pytest not in $WORKTREE_PATH/.venv — re-run 'uv sync --extra dev' manually"
```

`--extra dev` installs from `[project.optional-dependencies].dev` (pytest/mypy/ruff/pytest-cov); `--group dev` installs from `[dependency-groups].dev` (subset). Dropped the `2>/dev/null` muffle so real sync failures are visible. Added a WARN probe for pytest as a belt-and-suspenders check.

### T-251 — slash-normalization, end-to-end

`plugins/cloglog/skills/demo/SKILL.md` templates (both backend and frontend variants) composed `DEMO_FILE` from raw `$(git rev-parse --abbrev-ref HEAD)`. On slash-named branches (`feat/foo`), the template wrote `docs/demos/feat/foo/demo.md`, but `scripts/check-demo.sh` normalizes via `${FEATURE//\//-}` and looks in `docs/demos/feat-foo/` — silent `make demo-check` failure.

First-cut fix patched only the templates. Codex reviewer caught that this was a half-fix — `scripts/run-demo.sh:27,41` uses raw `$FEATURE` with `grep -qi "$FEATURE"` for its directory lookup, so `make demo` would still fail to *find* the script even when the template wrote the demo to the right place. Second commit added `FEATURE_NORM="${FEATURE//\//-}"` in `run-demo.sh` and switched the lookup to use it. Now the writer (SKILL.md), the discover step (run-demo.sh), and the verifier (check-demo.sh) all agree.

### T-252 — durable truth about `mcp-server/dist/`

Grepped the tree for "`they are checked in`" and "`commit the resulting .* mcp-server/dist`". The misstatement lived only in the T-247 work-log (historical record). No live template carried it — so this was the outcome-2 path: add a durable note to CLAUDE.md.

First-cut wording claimed `dist/` is "rebuilt by `on-worktree-create.sh` / CI". Codex reviewer flagged this as factually wrong — `.cloglog/on-worktree-create.sh:27-29` only does `npm install` (and only on `wt-mcp*` worktrees), and `.github/workflows/ci.yml:74-75,95-96` runs `npm ci` + tests + contract-check with no `npm run build`. Reviewer was correct; I had conflated "installs deps" with "rebuilds". Replaced with the actual truth:

> **`mcp-server/dist/` is gitignored — you rebuild it, nothing else does.** It is listed in both the repo-root `.gitignore` and `mcp-server/.gitignore`. Do NOT commit `dist/`. Neither `on-worktree-create.sh` (only `npm install`s on `wt-mcp*` worktrees) nor CI (`.github/workflows/ci.yml` runs `npm ci` + tests, no `npm run build`) rebuilds it. If your change touches `mcp-server/src/`, run `cd mcp-server && make build` yourself — local MCP clients consume `mcp-server/dist/index.js` directly, and `scripts/demo-update-task-status.mjs` imports from `../dist/server.js`, so stale artifacts bite at runtime.

Also simplified the pre-existing "sync with `--extra dev`, not `--group dev`" bullet per the task prompt, since the bootstrap script now does the right thing.
