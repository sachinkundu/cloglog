## Problem

`_build_body_header` rendered `**codex — turn {turn}/{max_turns}**` where `max_turns` was the intra-session cap (`codex_max_turns=2`, `opencode_max_turns=5`). Every webhook firing starts a fresh session at turn 1, so every session's first codex comment read `turn 1/2`. Observed on PR #209: two codex reviews both labelled `turn 1/2` with no way for a reader to tell session 1 from session 2.

Since T-227 switched the stop condition to approval-based stop + 5-session backstop, the intra-session turn index no longer governs any reader-actionable decision. The counter that matters is the cross-session one (1..`MAX_REVIEWS_PER_PR=5`).

## Learnings

### `ast.unparse` substring checks need structural scoping

`ast.unparse(Return_node)` prepends the `return` keyword, which contains `"turn"` as a substring — causing false positives when checking that a return value doesn't contain `"turn"`. Always unparse `ret.value` (the expression node), not the wrapping statement. Apply to `ret.value`, `fn.args`, `node.test`, and similarly scoped nodes.

### `uv run --quiet python` for demos that import project modules

Demos that stay stdlib-only (`ast`, `pathlib`, `json`) can use plain `python3 - <<PY`. Demos that must import project code (proof-by-direct-call) need `uv run --quiet python - <<PY` — `--quiet` suppresses uv's progress noise so captured stdout is deterministic.

### Plain Python `import` does not trigger `conftest.py` loading

`python -c "from tests.gateway import test_review_loop as T; T.TestX().test_y()"` imports only the target module — pytest's conftest auto-discovery is a pytest-runtime feature. This is a clean Showboat verify pattern for running specific test methods without fixture side-effects (e.g., the session-autouse Postgres fixture in `tests/conftest.py`).

### Opencode-only host has a hard constraint on `count_bot_reviews`

`TestOpencodeOnlyHost::test_session_cap_check_skipped_when_codex_unavailable` pins that `count_bot_reviews` MUST NOT be called when `_codex_available=False`. Any future plumbing needing a prior session count must gate the HTTP call on `_codex_available` or pre-seed a fallback (e.g., `prior = 0` before the capability-gated block). When hoisting logic out of a capability gate, check for regression-guard tests that pin "this HTTP call is NEVER made in mode X".

### Fast-forward from `origin/main` before any diff-based tool

After worktree creation, if the local branch is behind `origin/main`, `git diff origin/main --stat` shows phantom additions (the missed commits). Run `git merge --ff-only origin/main` before running the demo classifier, PR-body drafting, or any diff-based check — restores an accurate baseline.

### Required keyword-only constructor args as a plumbing firewall

Adding new load-bearing values as keyword-only args with no defaults (after `*,`) makes the type checker + mypy raise TypeError at construction time — not silently at runtime when the value is first used. Prefer required keyword-only args over `Optional`/`None`-sentinel defaults when the caller always has the value and it's critical to the class contract.
