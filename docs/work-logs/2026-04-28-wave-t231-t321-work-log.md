## Learnings & issues

- **`protect-worktree-writes.sh` scope keys must match the launcher's `wt-<scope>` convention.** The hook strips `wt-` from the worktree basename and looks the remainder up in `worktree_scopes` (with prefix matching: `wt-frontend-auth` → `frontend`). A live mapping whose keys don't match the launcher's actual worktree names leaves the hook in allow-all mode silently. **Init can't auto-generate the live mapping** because it has no way to predict the operator's launch convention; the only honest default is a commented template. The right long-term design (filed as residual): `/cloglog launch` writes an explicit scope identifier (file or env var) when it materializes a worktree, and `protect-worktree-writes.sh` reads that instead of basename-stripping.
- **A pin test that isn't in `make invariants` doesn't count as an invariant.** The framework's whole point is fail-fast; a regression test that only runs in the broader `make quality` test phase still catches the bug eventually but defers detection past the documented gate. New silent-failure pin tests must be added to BOTH the `Makefile` `invariants` recipe AND `docs/invariants.md` in the same PR — codex round 1 catches this every time.
- **Makefile recipes can't take inline `#` comments inside `\`-joined lines.** Place explanatory comments as Make-level blocks above the recipe, not interleaved with the shell continuation. Cite the task + incident so the rationale survives a future cleanup pass.
- **Pin tests on multi-occurrence flags must resolve line continuations.** A substring match against the file as a whole passes when only one of two invocations carries the flag. Parse each invocation block (joining `\` continuations) and assert per-block.
- **First codex round on protocol/contract changes catches the structural bug.** When changing a config schema that another component reads, audit the consuming component before writing tests — codex will.
